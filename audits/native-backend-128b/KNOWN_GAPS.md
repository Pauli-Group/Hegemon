# Native Backend 128-Bit Known Gaps

1. **This is not a Neo/SuperNeo implementation.** The backend provides commitment binding and a structural folding skeleton over Goldilocks. It does **not** implement the sum-check interactive reduction (Π_CCS), the random-linear-combination reduction (Π_RLC), the decomposition reduction (Π_DEC), or any CCS constraint-satisfaction check during folding. Folding verifies only that commitments were linearly combined correctly under the Fiat-Shamir transcript; it does not verify that the underlying witness satisfies the CCS relation. Soundness on the live tx-leaf path comes from the STARK-verification gate at leaf construction and at leaf import, not from the folding backend. Anyone evaluating this package must understand that the folding layer alone does not provide knowledge soundness over the CCS relation.
2. The active family now defines the exact bounded-kernel Module-SIS reduction for the implemented message class and computes one explicit coefficient-space Euclidean SIS estimate for the live instance, but that concretization and estimator choice still need independent external review and cryptanalysis. In particular, the active ring `Z_q[X]/(X^8 + 1)` is fully splitting over the Goldilocks field (since `q ≡ 1 mod 16`), so the Module-SIS instance flattens to unstructured SIS in coefficient space. The in-repo claim already operates at the flattened coefficient level, but external reviewers should verify that the splitting does not enable algebraic attacks below the stated 865-bit quantum floor.
3. The active floor is still tied to the explicit bounded-message cap `max_commitment_message_ring_elems = 513` and the explicit receipt-root cap `max_claimed_receipt_root_leaves = 128`; if either cap grows, the floor must be recomputed and may fall.
4. The timing harness is a regression screen, not a proof of constant time.
5. The review package prepares external cryptanalysis; it does not replace external cryptanalysis.
6. The chain architecture still has linear receipt-root verification and does not solve cold import.
